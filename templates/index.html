Model Architecture & Dataset

Algorithm
⚡
XGBoost

A gradient-boosted decision tree ensemble with 5000 trees (max depth 10), GPU-accelerated training and inference. Ultra-low threshold (0.001) catches 89.6% of attacks while maintaining a low false alarm rate.

Dataset Used
📊
NSL-KDD

Trained on the benchmark NSL-KDD dataset. It evaluates 41 independent network packet features (e.g., protocol_type, src_bytes, failed_logins) to classify behavior.

Test Accuracy
🎯
90.6%

Achieved 90.57% accuracy on the NSL-KDD test set. Catches 89.6% of attacks with only 8.2% false alarms using an ultra-low decision threshold of 0.001.

Preprocessing Pipeline

// Pipeline Steps: 1. Data Ingestion: Raw packet features received via REST API. 2. Feature Engineering: 17 derived features added (ratios, interactions, scores). 3. Categorical Encoding: 'protocol_type', 'service', 'flag', 'protocol_service', 'protocol_flag' → One-Hot Encoding (~220 features). 4. Feature Alignment: Re-indexed against training columns (missing columns zeroed out). 5. Binary Inference: XGBoost GPU with threshold=0.001 → 'normal' or 'attack'. 6. Attack Type: Multi-class XGBoost GPU classifies into DoS/Probe/R2L/U2R.

All incoming requests are scrubbed and aligned with the exact feature matrix shape that the model expects, ensuring seamless deployment and preventing input errors.
